# /report - 生成项目报告

生成项目进度报告

## 使用方式

```
/report daily     # 日报
/report weekly    # 周报
/report [项目名]  # 指定项目报告
```

## 日报模板

```markdown
# 项目日报 - YYYY-MM-DD

## 今日完成
### 设计
- [x] [任务]

### 开发
- [x] [任务]

### 测试
- [x] [任务]

## 进行中
- [ ] [任务] - [负责人] - [预计完成时间]

## 阻塞问题
| 问题 | 影响 | 解决方案 | 负责人 |
|------|------|----------|--------|
| [问题] | [影响范围] | [方案] | [人] |

## 明日计划
1. [计划]
```

## 周报模板

```markdown
# 项目周报 - 第X周 (MM-DD ~ MM-DD)

## 本周概览
| 类型 | 计划 | 完成 | 完成率 |
|------|------|------|--------|
| 设计 | X | X | XX% |
| 开发 | X | X | XX% |
| 测试 | X | X | XX% |

## 里程碑进度
| 里程碑 | 目标日期 | 当前状态 | 风险 |
|--------|----------|----------|------|
| [名称] | [日期] | XX% | 低/中/高 |

## 本周亮点
1. [亮点]

## 问题与风险
| 问题 | 状态 | 影响 | 应对措施 |
|------|------|------|----------|
| [问题] | 已解决/进行中 | [影响] | [措施] |

## 下周计划
### 设计
- [ ] [任务]

### 开发
- [ ] [任务]

### 测试
- [ ] [任务]

## 需要决策
1. [需要用户决策的事项]
```

## 存储位置

报告存储到 `../shared/tasks/reports/`

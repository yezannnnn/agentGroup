# /status - 团队状态汇总

查看团队各成员工作状态

## 使用方式

```
/status           # 查看全部状态
/status design    # 查看设计状态
/status dev       # 查看开发状态
/status test      # 查看测试状态
```

## 执行步骤

### 1. 读取共享状态
读取 `../shared/status.json`

### 2. 汇总各成员状态

```markdown
# 团队状态汇总 - YYYY-MM-DD HH:MM

## 设计 (艾拉)
| 任务 | 状态 | 更新时间 |
|------|------|----------|
| [任务名] | 进行中/已完成/待开始 | [时间] |

## 开发 (贾维斯)
| 任务 | 状态 | 更新时间 |
|------|------|----------|
| [任务名] | 进行中/已完成/待开始 | [时间] |

## 测试 (凯尔)
| 任务 | 状态 | 更新时间 |
|------|------|----------|
| [任务名] | 进行中/已完成/待开始 | [时间] |

## 待处理通知
[列出未处理的通知]

## 阻塞点
[如有阻塞，列出并建议解决方案]
```

### 3. 风险预警
如发现以下情况，主动提醒：
- 任务长时间无更新
- 依赖关系阻塞
- 资源冲突

# UI 结构化模板

> 将UI设计稿转化为此格式，供前端开发使用

---

## META
```yaml
页面: [页面名称]
设计师: [姓名]
日期: YYYY-MM-DD
尺寸: [设计稿宽度]px
链接: [Figma/蓝湖链接]
```

## 页面结构
<!-- 用树形结构描述布局 -->
```
页面
├── 头部 (Header)
│   ├── Logo [左, 120x40]
│   ├── 导航 [中, flex]
│   └── 用户信息 [右, 200px]
├── 主体 (Main)
│   ├── 侧边栏 [左, 240px, fixed]
│   └── 内容区 [右, flex]
└── 底部 (Footer)
    └── 版权信息 [居中]
```

## 组件清单

| 组件 | 位置 | 尺寸 | 状态 |
|------|------|------|------|
| 按钮-主要 | 表单底部 | 120x40 | default/hover/disabled |
| 输入框 | 表单 | 100%x40 | default/focus/error |
| 卡片 | 列表 | 280x160 | default/hover |

## 组件详情

### 按钮-主要
```yaml
尺寸: 120x40 | 圆角: 8px
字体: 16px medium #FFFFFF
背景: #1890FF
状态:
  hover: 背景 #40A9FF
  disabled: 背景 #D9D9D9, 字色 #999
```

### 输入框
```yaml
尺寸: 100%x40 | 圆角: 4px
边框: 1px #D9D9D9
字体: 14px #333
placeholder: #999
状态:
  focus: 边框 #1890FF
  error: 边框 #FF4D4F, 提示文字下方
```

## 交互说明

### 表单提交
```
点击提交
→ 校验失败 → 高亮错误字段 + 显示提示
→ 校验成功 → 按钮loading → 请求
  → 成功 → Toast提示 + 跳转
  → 失败 → Toast错误信息
```

### 列表加载
```
滚动到底部
→ 显示加载动画
→ 加载完成追加数据
→ 无更多数据显示"没有更多了"
```

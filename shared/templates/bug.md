# BUG 结构化模板

> 简洁高效的Bug报告格式

---

## BUG-[YYYYMMDD]-[序号]

```yaml
标题: [一句话描述]
级别: P0紧急|P1高|P2中|P3低
状态: 待修|修复中|待验|已关
发现: [姓名] @ YYYY-MM-DD
指派: [开发姓名]
```

### 现象
<!-- 发生了什么 -->
[简述问题现象]

### 复现
```
环境: [浏览器/系统/版本]
账号: [测试账号，如有]
步骤:
1. [操作1]
2. [操作2]
3. [操作3]
结果: [实际结果]
期望: [期望结果]
```

### 证据
<!-- 截图描述或日志关键行 -->
```
[错误日志/截图描述]
```

### 影响
```yaml
范围: [影响的功能/用户群]
频率: 必现|偶现(X/10)
```

### 修复
<!-- 修复后填写 -->
```yaml
原因: [根本原因]
方案: [修复方案]
文件: [修改的文件]
验证: [如何验证]
```
